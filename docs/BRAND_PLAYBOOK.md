# Chiti Technologies — Brand Playbook

## Core Positioning

| Attribute | Statement |
|---|---|
| **One-Liner** | We build intelligent systems, not just beautiful websites. |
| **Category** | Premium systems engineering & automation collective |
| **Not a** | Web agency, design studio, or IT outsourcing firm |
| **Pricing Anchor** | Minimum engagement ₹1,50,000 — strategic partnerships, not page counts |
| **Relationship Model** | Engineering partner, not vendor |

## Vocabulary & Voice

### Do Say

| Term | Context |
|---|---|
| Product Design | Replaces "UI/UX design" in headlines, roles, metadata |
| Interface Engineering | Technical case studies and capability descriptions |
| System Architecture | Primary service category umbrella |
| Automation & Workflow Systems | Second service category |
| Scalable Design Systems | Third service category |
| Intelligent Systems | Core brand promise — the headline verb |
| Digital Infrastructure | What we build (not "websites") |
| Engineering Partner | Relationship model (not "agency") |
| Digital Ecosystems | What we create for clients |
| Architectural | Adjective — our thinking is architectural, not cosmetic |

### Don't Say

| Avoid | Use Instead |
|---|---|
| UI/UX design (in headlines) | Product Design |
| Web agency / Digital agency | Systems collective / Engineering partner |
| Websites (as output) | Digital infrastructure / Systems |
| Web development (sole descriptor) | System architecture + engineering |
| Design studio | Engineering & product design studio |
| IT solutions / software development | Commoditized — avoid entirely |

### Canonical Service Names

1. **Identity Infrastructure** — strategic brand architecture with tokenized design systems, voice frameworks, multi-touchpoint governance
2. **Automation & Workflow Systems** — custom automation pipelines, CRM infrastructure, digital ecosystems that replace manual processes with intelligent orchestration
3. **Scalable Design Systems** — component-driven visual languages with semantic tokens, accessibility-first patterns, developer-ready implementation

### Pricing Narrative

- "Strategic partnerships, not page counts"
- "Minimum engagement: ₹1,50,000"
- "We evaluate every engagement by operational complexity, system architecture, and business goals — not by deliverables or page counts."

## Visual Hierarchy (Hero Headline)

The headline "We build intelligent systems, not just beautiful websites" uses controlled visual weight to match the brand message:

| Word | Treatment | Purpose |
|---|---|---|
| **intelligent** | Gradient primary→secondary, 1.15em, weight 900, breathing glow animation | **Focal point** — eye lands here first |
| **beautiful** | Muted `--color-on-surface-variant`, weight 500 | Visually subordinate — beauty is a given, not the sell |

This ensures the typographic emphasis matches the copy's claim: intelligence is premium; beauty is table stakes.

## Brand Territory

### We operate at the intersection of:

1. **System Architecture** — deep technical infrastructure, data modeling, integration layers
2. **Product Design** — user psychology, business workflows, interface logic, UX strategy
3. **Automation Engineering** — CRM pipelines, intelligent orchestration, workflow automation
4. **Digital Craft** — precision execution, editorial-grade interfaces, motion design

### We explicitly exclude:

- Social media management
- Content writing / SEO blogging
- Print / packaging design
- Low-volume template-based websites
- Staff augmentation / body shopping
- Traditional IT outsourcing

## Case Study Narrative Framework

Every case study follows this structure to reinforce the brand positioning:

1. **Problem** — operational friction, system bottleneck, business constraint
2. **System Overview** — architecture diagram, tech stack rationale
3. **Challenges** — technical complexity, scale constraints, integration hurdles
4. **Design Decisions** — why specific architectural / interface choices were made
5. **Screens** — editorial-grade interface showcase
6. **Features** — key capabilities delivered
7. **Impact** — metrics, performance gains, business outcomes
8. **Reflection** — architectural lessons, what we'd evolve
